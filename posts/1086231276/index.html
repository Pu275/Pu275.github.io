返回值类型

默认的返回策略是 py::return_value_policy::automatic。下面的例子，get_data 绑定到 Python 以后，默认的 automatic 返回策略会让 Python 接管 data 的所有权，之后的 C++ 数据 data 的销毁也由 Python 的机制负责。这样可能造成多次 delete，因为 C++ 可能已经写好了 data 的释放规则。

所以更好的写法是指出返回值类型为 reference，把销毁的工作交给 C++ 来做。

take_ownership：Python 负责调用

返回指针会怎么样

答案：会解引用，因为 Python 没有指针啊。

buffer protocol

我们希望避免 C++ 的数据传递到 Python 发生的拷贝，可是 C++ 直接提供一个数组指针 Python 是无法理解的（这个指针会被解引用），如果要把数组传递给 Python 需要进行绑定。

可以为 C++ 的数据类注册一个 view，通过 view 可以高效的把指针看管的数据交接给 numpy。这种方法传递的数据需要符合 Python 规定的 buffer protocol：

（1）起始指针 buf（2）元素大小 itemsize（3）对应的 Python 数据类型 format（4）维度数 ndim（5）维度 shape（6）各维度的步长 strides

STL 容器的转换（C++ 到 Python）

头文件 pybind11/stl.h 提供了 STL 容器到 python 内置数据结构的自动转换。缺点是每次 Python 端访问 STL 容器的时候都会发生拷贝（因为 Python 和 C++ 的相同数据结构可能在内存布局不一样）。另外，C++ 类里面的 STL 容器成员即使在绑定的时候设置了 readwrite 权限，Python 端访问这个成员依然是创造一个拷贝。

可是我们有些时候要修改 STL 容器，就要禁用自动转换的拷贝。重新注册一个 opaque 版本的容器类。pybind 提供了 make_opaque 宏，在绑定操作前面用 make_opaque 宏注册不需要拷贝的容器：

传入 numpy 数组

从 Python 到 C++ 的 buffer protocol，最常见的就是 numpy 数组了。C++ 用于承接的函数只需要 py::buffer 作为形参即可

注意： 传进来 buffer 的指针也是可以 free 掉的

pybind11::array_t

把 py::buffer 替换成 py::array 可以把 buffer protocol 对象的范围限定在 numpy 数组上，推荐使用。array_t 是 array 的模板版本，指定数据类型。py::array_t 作为返回值的类型默认是 reference

初始化

array_t 有很多初始化方式，基本构成是四个参数，dims、strides、pointer、pyobject

  1. dims、strides
  2. dims、strides、pointer、py::object
  3. buffer_info、py::object，用 buffer_info 里面的 dims、strides 信息

关闭 Python 到 C++ 的隐式转换

如果隐式转换不是必要的，可以在注册的时候关闭。在参数后面加上 noconvert。这样注册的函数就变成了严格类型模式

注意： 如果只有部分参数需要 noconvert，还是要把全部参数注册 py:arg 然后标出不需要隐式转换的
