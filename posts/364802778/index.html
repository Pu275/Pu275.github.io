    C++的STL容器

    发表于 2022-03-12 更新于 2022-05-11 分类于 C++ Valine：

    容器

    stack

    push： 推入元素

    top： 获取堆顶元素

    pop： 删除堆顶元素

    emplace： 传入对象的时候等价 push，传入参数是调用构造函数，而不是拷贝构造函数，可以节省内存

    queue

    front： 获取第一个元素

    pop： 第一个元素删掉

    back： 获取最后一个元素

    push： 尾巴加一个

    deque

    和 queue 的区别是，deque 是双端队列，可以从头部加元素。没有 top！

    push_back： 从后面加元素

    push_front： 从前面加元素

    pop_back： 从后面删除元素

    pop_front： 从前面删除元素

    empty、clear 也有

    array

    类似原生数组，固定长度

    list

    支持插入和移除的链表，不支持随机访问（因为链表内存是不连续的）。list 的优势是插入操作不会导致扩容，也就不会有 vector 因为扩容发生的拷贝。所以 list 适合存放复杂的数据。

    pair

    pair 是 tuple 的特例，只有一对元素，类型可以不同

    first、second： 可以修改元素
    std::get<0>： 也可以通过 std::get<下标> 来访问

    作为函数返回值

    使用 tie 解开 pair

    tuple

    和 pair 差不多，元素数目不受限，获取元素只能通过 std::get<0>。

    unordered_map

    insert： 插入，但不会覆盖已有的 value
    first、second： key 和 value

    begin()、end()： 迭代器

    查找 key

    自定义 key

    如果想用自己的类作为 key，需要为 key 指定 hash 函数 和 相等运算

    unordered_map 默认调用 std::hash 作为哈希函数，std::string 拥有自己的 std::hash 重载版本，所以可以直接作为 key。

    感觉很麻烦，也可以直接用类指针作为 key

    unordered_set

    用法类似 vector

    priority_queue

    优先队列： 任何时候弹出的都是优先级最高的元素（有点抽象）其实就是最大元素最先出来，即使他不是最早进去的

    堆： 采用树形结构实现优先队列的手段，大顶堆的插入只需要 log2Nlog_2Nlog2​N 次交换即可选出最大元素

    使用堆实现优先队列包含两个问题（1）插入元素如何重新堆化（2）弹出元素如何重新堆化

    （1）在尾部加入元素，往上不断的比较他的父结点（下标 (i - 1) / 2），比到合适的交换即可。（不一定要一次次交换上去）

    （2）删除头部元素：把末尾元素放到堆顶，然后从头部做一次向下筛选即可。（除了头部以外，其他保留着堆序）

    C++里面优先队列当做堆使用就行，默认大顶堆

    push、pop、top、size 操作类似 queue

    泛型算法

    标准库算法使用 begin（头元素）和 end（尾元素后一位）迭代器。泛型指的是同一个名称的算法可以应用在各种容器上面。

    大部分算法在 algorithm，数值相关的在 numeric。算法不依赖容器，但是依赖运算符，比如 find：==
