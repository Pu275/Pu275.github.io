一个创建类的例子

一个 Pet 类，有自己的名字变量 name。

完成他的绑定，必须要注册构造函数 py::init。所有暴露给 Python 的成员都要经过绑定，即使 public 成员变量默认是不可读的，需要绑定成 “def_readonly” 或者 “def_readwrite”

构造函数

  1. 用 py::init<形参> 绑定，最简单常用的方法
  2. 把一个返回 new 出来的类指针的匿名函数，传给 py::init。据说推荐用 std::unique_ptr
  3. 定义一个 static 返回类实例的工厂函数，并且绑定

private 成员变量 name 就没法 只读 或者 读写 绑定了，但是可以设置两个函数来实现这个操作（读取，赋值），绑定为 “def_property”（类似 python 的 @property 方法）

动态属性

C++ 定义的类默认不支持动态添加类成员，唯一可写的属性是使用 def_readwrite 或者 def_property 定义的属性。

除非我们开启 “py::dynamic_attr”

注意： 具有动态属性的 C++ 类会带来一些运行时开销。 默认情况下，pybind11 类比原生 Python 类更高效。 启用动态属性只会使它们处于同等水平。

绑定的时候不能直接照搬 Pet 的流程做一个 Dog 出来，需要特别指出 Dog 的基类是什么。下面两个方法是等价的。

即使 Python 用不到 Pet 这个类，还是得完成他的绑定，Dog 才能在 Python 里面使用 Pet 的成员

如果 Pet 不绑定，Dog 却绑定了，Python 导入文件直接会报错。如果 Pet 没有注册方法，那么 Dog 相应的也拿不到继承来的权限。

换句话说，不管类是怎么定义的，绑定阶段重新分配了那些是可见的对象。

如果 Pet 把 name 设置成 private，Dog 想要访问 name 也得用 “def_property”，类似 Pet 设置

数据结构转换

头文件 pybind11/stl.h 提供了 python 内置数据结构到 stl 容器的自动转换。缺点是每次自动转换都会发生拷贝，因为即使 Python 和 C++ 的相同数据结构也可能在内存又不一样的布局。如果要禁用拷贝，需要重新注册一个 opaque 版本的容器类。

可是我们会需要修改传入的参数，解决方案是把要修改的数据封装容许修改的容器里面，下面的 buffer protocol 就是

buffer protocol

为了避免 C++ 到 Python 之间数据的拷贝，可以为 C++ 的数据类注册一个 view，通过 view 可以高效的把指针看管的数据交接给 numpy。这种方法传递的数据需要符合 Python 规定的 buffer protocol：（1）起始指针 buf（2）元素大小 itemsize（3）对应的 Python 数据类型 format（4）维度数 ndim（5）维度 shape（6）各维度的步长 strides

从 Python 到 C++ 的 buffer protocol，最常见的就是 numpy 数组了。C++ 用于承接的函数只需要 py::buffer 作为形参即可

注意： 传进来 buffer 的指针也是可以 free 掉的

pybind11::array

把 py::buffer 替换成 py::array 可以把 buffer protocol 对象的范围限定在 numpy 数组上

关闭 Python 到 C++ 的隐式转换

如果隐式转换不是必要的，可以在注册的时候关闭。在参数后面加上 noconvert。这样注册的函数就变成了严格类型模式

注意： 如果只有部分参数需要 noconvert，还是要把全部参数注册 py:arg 然后标出不需要隐式转换的
