整理一些习题

6.36

编写一个函数声明，使其返回数组的引用并且该数组包含10个string对象。不用使用尾置返回类型、decltype或者类型别名。

为上一题的函数再写三个声明，一个使用类型别名，另一个使用尾置返回类型，最后一个使用decltype关键字。你觉得哪种形式最好？为什么？

6.39

说明在下面的每组声明中第二条语句是何含义。如果有非法的声明，请指出来。

（a）：不合法，第二条语句可以接收 int 或者 const int 类型，而且函数体内不可以对变量赋值。第一条语句也可以接收 int 或者 const int，传参的时候两种声明都匹配，编译器没法识别调用的是哪一种。

（b）：不合法，两个声明参数相同，编译器没法分辨

（c）：合法。接收 double 指针，返回 double 指针

6.53

说明下列每组声明中的第二条语句会产生什么影响，并指出哪些不合法（如果有的话）。

（a）合法，整数引用的底层 const，不能修改引用的本体

（b）合法，char 指针的底层 const，不能修改指针指向的内存

（c）不合法，char 指针的顶层 const，函数内部指针值不会受到影响，所以加不加顶层 const 都是一样的（换句话说，函数传参指针不需要顶层 const ？就算加了也会被编译器理解成一件事）

小结

传参的时候，以下情况是容易产生二义性的。而且函数传值会进行拷贝，所以设置只读参数的意义也不大。

传值的场合不需要加 const
