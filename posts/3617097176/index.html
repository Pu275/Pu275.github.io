Python 的引用

常量们

Python 有一个小整数池 [-5, 256] 提前在内存中分配好，任何变量赋值同一个小整数，都是对小整数池的引用。

新建了一个大整数，会在大整数池添加，如果之后再用这个变量赋值给其他变量，也是赋值引用。

字符串、浮点也和大整数池一样，如果重新赋值就是引用。

可变对象们

赋值都是引用，传参也都是引用。Python 对象大部分都是引用

切片，情况变得复杂

总体上列表的切片可以看作 浅拷贝 。可是虽然 列表的切片 产生了新的对象，但它的元素是对原列表元素的引用！而且对切片的修改不会作用到本体上

是这样一个思路： 切片产生的是原列表元素的引用，类似函数传参，如果是可变对象（列表、字典），修改会作用在本体上；如果是不可变对象，修改不会影响到本体，只是拷贝的元素地址发生变动。

什么是引用计数

知道了常量的引用规则，可以看一个最简单的引用计数例子了。Python 分配的内存上会统计指向该内存变量的个数，变量指向其他内存，计数减一；新变量指向这块内存，计数加一。如果计数归零，说明内存没用了，即可释放。

引用计数的目的就是清理垃圾，腾出没用的内存空间。

本来为 x = 1.2321 分配的内存在 x 删除以后，依然会保留下来（因为 y 还在引用它）。如果最后删除 y，这块内存会 无人看管吗 ？不会，因为此时它的引用计数归零了，Python 的自动回收机制避免了内存泄露的问题。

不可变对象

“不可变” 指的到底是什么？对象引用的内存里的值不可变。不可变对象也是传递引用，之所以副本的改动不会影响本体，是因为，不可变对象的赋值和传参，会在内存重新发生构造，并且传递新引用。（而不是在原位直接修改）

不可变对象传参传的什么

不可变对象 y，确实是把 y 的引用传入了函数 add，但是发生了赋值操作以后，y 就绑定到新的整数地址上去了。结果上，传进去的 y 本体没有发生改变。虽然在函数内部它的值变了，只是因为它临时指向了另一块地方

不可变对象 x 的重新赋值，先为 x + 1 的值在内存腾出空间，然后 x 改成引用这块内存。

浅拷贝和深拷贝

copy：浅拷贝，deepcopy：深拷贝。二者的区别在于，浅拷贝只拷贝“指针值”，导致副本可以修改“指针对象”，而深拷贝会把指针的内容进行拷贝，导致副本和本体彻底独立

命名空间和作用域

命名空间

局部命名空间： 函数内变量、参数，优先级最高

全局命名空间： 函数外（模块内）变量，包含其他文件导入的模块内变量，优先级中

内置命名空间： 内置方法、名称，优先级最低

作用域

Local： 最内层的函数，优先级最高

Enclosing： 内层和全局中间的作用域

Global： 函数或者类的外部

Built-in： 内建变量，优先级最低

模块、类、函数会创建新的作用域。比如说，函数内部的作用域（local）变量，在函数外部（global）就没有权限获得

全局变量

对于不可变对象，函数内部的修改不会作用到外部

对于可变对象，函数内部的，作用在内存上的修改可以影响到外部

赋值操作会导致对外部对象的引用丢失。除非函数加上 global 关键字声明

对象占用的字节数

sys.getsizeof 可以获得对象占用的字节数：

C++ 里面 int 只有 4 字节。一个字节 8 bit，所以 C++ int 的范围是 -2^31 到 +2^31。为什么 Python 的 int 反而比 float 占用更多？而且这么多。据说是因为 Python 的 int 没有上下限，可以不断扩容。可以看到在 2^30 的分界线发生了扩容 4 字节（为什么不是 2^31 呢）

del 和 __del__ 的区别

del 是将对象的引用计数 -1，__del__ 是定义了这个方法的类在引用计数归零的时候调用的方法。

view

view 类似浅拷贝，但又不是浅拷贝。不仅元素的地址被拷贝过去，而且 view 对元素的修改也会同步在本体上。另一方面，view 也会使得本体变量的引用计数 +1，所以只有在 view 和本体都没了以后内存才会释放
