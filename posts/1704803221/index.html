不简单的排序

发表于 2022-03-08 更新于 2022-04-14 分类于 排序 Valine：

本篇包含：

  • 快速排序
  • 归并排序
  • 计数排序
  • 堆排序

快速排序

思路：选取枢轴量 pivot，小的元素挪他左边，大的挪他右边，然后对左右两边重复此过程。

归并排序

思路：合并有序数组效率比较高，把原数组细分成最小粒度的有序子数组，再合并他们

注意： 原位 merge 很难（估计也没法实现），还是要额外建一个 vector

混合版归并排序

可以设定归并的阈值，比如说 20 个元素为一组，局部用插入有序，最后合并

计数排序

思路：统计最小值到最大值每个整数的出现次数，然后从小到大放回数组。

桶排序

思路：类似归并排序，只不过几个桶本身已经有序（区间1 < 区间2），桶里面排好序即可。计数排序本质上是一种特殊的桶排序（一个整数一个桶）

场景：上下界已经确定，就可以划分区间了

堆排序

堆是什么

  1. 堆是完全二叉树
  2. 堆的每个结点都小于（或大于）父结点的值

堆排序是什么

  1. 构建大顶堆（根结点是最大元素的堆）
  2. 交换根结点和最后一个结点，最大元素移到数组末尾固定下来
  3. 继续重复构建前 n−1n-1n−1 个元素的大顶堆，直到只剩下根结点没有固定

和选择排序的区别

选择排序的每轮迭代也是从剩下的元素中挑出最小的那个固定下来，堆排序的优点在于，每次迭代会利用到之前的信息，因为建堆完后的二叉树从上到下可以保证大体上是由大到小的，而且顶顶的肯定是最大的。在每次首尾交换之后，重新恢复建堆只需要大约 log(n)log(n)log(n) 次交换，相比选择排序需要扫描所有元素。

建堆

从最后一个非叶子结点（父结点，或者说有叶子的结点）开始，把它和它的左右结点（如果有）中最大的放到当前位置上。执行这个过程直到根结点

  • 如果根结点下标从 0 开始记，当前结点如果是 iii，那么它的左结点是 2i+12i+12i+1，右结点是 2i+22i+22i+2
  • 如果根结点下标从 1 开始记，当前结点如果是 iii，那么它的左结点是 2i2i2i，右结点是 2i+12i+12i+1

以上关系表明，获取子结点的父亲的办法是：

  • 如果根结点下标从 0 开始记，当前结点减 1 整除 2（也就是，下标 1 版本需要减 2 再整除 2）
  • 如果根结点下标从 1 开始记，当前结点整除 2

备注： 非叶子节点：0 到 length / 2 - 1。

heapify

一个递归函数，输入一个非叶子结点的下标，选取他的最大的儿子进行交换，并且追踪迭代换下去的父结点是否可以进一步交换（确保整个堆呈现一个 大的元素在顶上 的趋势）

建堆的过程就是从最后一个非叶子结点依次序执行 heapify 到根结点。

排序

完成初步建堆以后，每次把最大的元素放到尾巴，剩下的 n-1 个元素继续堆化。但是此时 n-1 个元素已经基本堆化了，我们只需要对根结点 heapify，就可以让调上来的元素跑到他该去的地方。（此时的调换次数少于选择）

堆的其他应用（TopK）

建一个 K 大小的堆可以把原本 nlogn 的排序升级成 nlogK 时间复杂度

以求解前 K 个大的元素为例：

  • 建立小顶堆，一旦新元素大于堆首，替换元素并重新建堆。
  • 也可以往堆里面不断加元素，如果长度超出 K 就移除元素并重新建堆

heapq

heapq 是 python 内置的堆包。

heapq 方法只接受 List 作为参数，除了 nsmallest 可以是可迭代对象
默认只提供了小顶堆，也就是 heap[0] 一定是列表最小的元素

heappop、heappush 都必须作用在 heap 上

siftdown 和 siftup

siftdown 需要给定起始点和终止点，把终止点的浮到合适的位置

siftup 给定起始点，把该元素下沉到合适的位置

注意： up 还是 down 有不同的说法，这里参考 heapq 的记法，从上到下记为 up，从下到上记为 down

heappop

把堆的首位元素弹出，并重新建堆（执行一次 siftup 即可）。

细节上，是把最后一位的元素弹出，并替换掉第一位的。之后的建堆，这个元素会沿着一条路径移动到合适的地方。如果直接弹出第一位元素，然后第二位作为第一位，这样无法保证得到的列表是一个堆（堆的结构被破坏了）

heappush

在堆的尾部加上新元素，并重新建堆（执行一次 siftdown 即可）。不用担心堆的结构改变了，只需要沿着新元素的路径走一遍 siftdown 即可。

heapreplace

替换堆的首位元素，执行一次 siftup

heapq 实现的堆排序

  1. 先 heapify 一次，然后新建一个 List，把堆首元素不断的 heappop 出来加入新 List 即可
  2. 传统思路，需要一个可以限定范围的 siftup 函数。因为是小顶堆，得到的是从大到小，最后要逆序一下
    heapify 一次，然后不断的把最小元素挪到 List 尾巴，并收缩 siftup 的范围
